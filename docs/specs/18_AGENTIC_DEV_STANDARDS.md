## 1. Purpose

This document defines the canonical rules, standards, and best practices governing all agentic development on the Lastmile Gig platform. It is the binding specification for any human developer, AI coding agent, or autonomous workflow that writes, modifies, or deploys code within the Lastmile Gig ecosystem.

Every AI coding agent operating in this codebase must:
1. Read and conform to this document before generating any code
2. Reference the relevant module spec (docs 01–19) before implementing any feature
3. Never bypass a HITL gate
4. Never write to production infrastructure directly — all changes via Git + ArgoCD

## 2. The Twelve Commandments of Agentic Development

### COMMANDMENT 1 — Read Before You Write
Before generating any code for a module, the agent must read:
- `01_MASTER_OVERVIEW.md` (platform context)
- The relevant module spec (e.g. `14_RESTAURANT_STOREFRONT_SPEC.md`)
- `02_TECH_STACK.md` (language and framework constraints)
- `10_SECURITY_COMPLIANCE.md` (security non-negotiables)

Generating code without reading relevant specs is a violation.

### COMMANDMENT 2 — One Phase, One Atomic Change
Every code generation task corresponds to exactly one phase in `17_DEVELOPMENT_ROADMAP.md`. No agent may bundle multiple phases into a single change. Each phase must be independently testable and independently deployable.

```
CORRECT:   Phase 015 → Add Supabase project initialisation and base schema
INCORRECT: Phases 015–020 → "Set up all databases at once"
```

### COMMANDMENT 3 — Language Discipline
No agent may select a language or framework not specified in `02_TECH_STACK.md` for a given service domain. The polyglot architecture is deliberate — every choice is documented and justified.

```
CORRECT:   Dispatch engine implemented in Go
INCORRECT: Rewriting dispatch engine in Python "because it's easier"
```

### COMMANDMENT 4 — Type Safety is Non-Negotiable
All TypeScript code must be written with `strict: true`. No `any` types. No `@ts-ignore`. No implicit `any` function parameters.

```typescript
// CORRECT
async function getDriver(id: string): Promise<Driver> { ... }

// VIOLATION — agent must refuse to generate this
async function getDriver(id: any): Promise<any> { ... }
```

### COMMANDMENT 5 — Security Controls are Immutable
No agent may generate code that:
- Bypasses JWT validation on any protected endpoint
- Stores secrets in code, environment variable files committed to Git, or any location other than HashiCorp Vault or AWS Secrets Manager
- Disables RLS on any Supabase table
- Removes WAF rules or security headers
- Logs PII (names, email addresses, phone numbers, biometric data) to any logging system

### COMMANDMENT 6 — All Endpoints Must Be Validated
Every API endpoint that accepts external input must have:
- Input validation using `class-validator` (NestJS) or equivalent
- Response serialisation using `class-transformer`
- Rate limiting via Upstash Redis middleware
- Auth guard (JWT or API key) unless explicitly documented as public

No agent may generate an unvalidated, unauthenticated endpoint without explicit documentation justifying it.

### COMMANDMENT 7 — HITL Gates are Sacred
No agent may remove, bypass, or short-circuit a Human-in-the-Loop gate defined in `05_AI_AGENTIC_LAYER.md`. When in doubt about confidence thresholds, err toward requiring human review.

```python
# CORRECT — HITL gate preserved
if confidence < 0.90:
    return await route_to_human_review(state, confidence)
return await execute_automatically(state)

# VIOLATION — agent must never generate this
# (removing HITL "for efficiency")
return await execute_automatically(state)  # always auto
```

### COMMANDMENT 8 — Tests Are Not Optional
Every function, service method, and API endpoint generated by an agent must have a corresponding test:
- Unit test (Jest / Pytest / Go test / ExUnit)
- Integration test for any database interaction
- E2E test for any user-facing flow (Playwright / Cypress)

Minimum coverage target: **80% per service, 95% for payment and blockchain services.**

### COMMANDMENT 9 — OpenTelemetry Instrumentation is Mandatory
Every new service, route handler, Kafka consumer, and background job must emit:
- A trace span with relevant attributes
- Prometheus metrics (request count, duration, error rate)
- Structured log entry with `trace_id` correlation

```typescript
// CORRECT — every route handler is instrumented
@Get(':id')
@UseInterceptors(OtelInterceptor)
async getOrder(@Param('id') id: string): Promise<OrderDto> {
  const span = trace.getActiveSpan();
  span?.setAttribute('order.id', id);
  return this.orderService.findById(id);
}
```

### COMMANDMENT 10 — Database Changes Require Migrations
No agent may alter a database schema without:
1. Writing a versioned migration file (Supabase migrations / Flyway for Java services)
2. Ensuring the migration is backward-compatible (no dropping columns without a deprecation phase)
3. Adding RLS policies for any new table
4. Updating the relevant schema section in `07_DATABASE_ARCHITECTURE.md`

### COMMANDMENT 11 — Infrastructure Changes via Terraform Only
No agent may provision, modify, or destroy any AWS resource except via Terraform. No AWS console changes. No AWS CLI `create` commands outside of Terraform. All Terraform plans must pass Checkov scan before apply.

### COMMANDMENT 12 — POPIA Compliance is a Hard Stop
If any generated code would result in:
- PII being stored without explicit consent
- PII being transmitted outside of `af-south-1` (except DR to `eu-west-1`)
- Biometric data being stored outside HashiCorp Vault
- Audit log entries being suppressed or deleted

...the agent must **halt, raise a compliance flag, and escalate to human review**. No exceptions.

## 4. Agentic Workflow Protocol

When an AI coding agent (Claude, Cursor, GitHub Copilot Workspace, or custom CrewAI dev agent) works on this codebase:

### 4.1 Start-of-Task Protocol
```
1. Read MASTER_OVERVIEW.md → confirm platform context
2. Read DEVELOPMENT_ROADMAP.md → identify current phase
3. Read relevant module spec → understand feature requirements
4. Read TECH_STACK.md → confirm language/framework for this service
5. Read SECURITY_COMPLIANCE.md → note any security requirements for this feature
6. Identify files to create/modify → confirm with human before proceeding
7. Write implementation → write tests simultaneously (TDD preferred)
8. Run linter, formatter, tests locally → confirm all pass
9. Raise PR with standard format → await human review
```

### 4.2 Uncertainty Protocol
If the agent is uncertain about:
- Which language to use → ask human, reference TECH_STACK.md
- Whether a feature requires HITL → default to YES, escalate
- Whether a database change is backward-compatible → ask human
- Whether an API change is breaking → ask human

**When in doubt, ask. Never guess on security, compliance, or breaking changes.**

### 4.3 Prohibited Agent Actions
The following actions are **strictly prohibited** for any agentic workflow:

```
❌ Direct AWS console or CLI resource creation
❌ Merging own PRs (agent cannot self-approve)
❌ Deploying to production (ArgoCD gates exist for a reason)
❌ Generating or handling private keys, wallet seeds, or secrets
❌ Modifying Terraform state directly
❌ Dropping or truncating any database table
❌ Removing or weakening any authentication or authorisation guard
❌ Bypassing RLS policies
❌ Writing PII to logs
❌ Disabling observability instrumentation for "performance"
```

### 5.2 Architecture Decision Records (ADRs)
Every significant architectural decision must be documented as an ADR in `/docs/adr/`:

```markdown
# ADR-001: Use Elixir/Phoenix for Real-Time WebSocket Service

**Status:** Accepted
**Date:** 2026-01-15
**Deciders:** Fortune (MD), Tech Lead

## Context
The platform requires persistent WebSocket connections for 10,000+ concurrent
driver location broadcasts. Node.js EventEmitter approach tested at 3,000 connections
showed memory issues. Go goroutines viable but lack the supervisor/recovery model needed.

## Decision
Use Elixir + Phoenix Channels. The BEAM VM handles millions of lightweight
processes with built-in fault tolerance via OTP supervision trees.

## Consequences
+ Handles target scale with <2MB RAM per 10k connections
+ Self-healing via supervision trees
- Adds Elixir to the language portfolio (training required)
- Smaller talent pool in South Africa
```
